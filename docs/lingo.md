# SAJUX Language Guidelines (Draft)

## 목표
- 내부 계산(전통 용어)은 유지하되, 사용자에게는 이해하기 쉬운 '상담가 톤'으로 전달합니다.  
- 모든 문장은 주어(누가)·목적어(무엇에 대해)·동사(무엇을 권하는지)를 분명히 합니다.

## 금지어 목록 (출력에 절대 노출 금지)
- 공망, 월주, 일주, 시주, 십성, 용신, 기신, 지지, 천간 (등 전통 용어)

## 허용 표현 (사용자에게 보여질 언어)
- 부모님과의 관계, 배우자와의 관계, 자녀와의 관계, 재정(돈)에 대한 불안, 일·직장 문제, 건강 관련 피로 등

## 문장 템플릿 (우선순위)
1. 공감 문장 (한 문장): "당신은 [대상]에서 가끔 허전함을 느낄 수 있습니다."
2. 구체적 느낌(사례): "'사랑을 못 받은 것 같아요' 또는 '거리감이 있어요'처럼 느껴질 수 있습니다."
3. 위로 문장(한 문장): "그런 감정이 있어도 당신의 가치는 변하지 않습니다."
4. 행동 권장(한 문장, 구체): "오늘은 짧은 안부 문자 한 통을 보내보세요."

문장 구성 예:
```
당신은 부모님과의 관계에서 가끔 허전함을 느낄 수 있습니다. '사랑을 못 받은 것 같아요' 또는 '거리감이 있어요'처럼 느껴질 수 있습니다. 그런 감정이 있어도 당신의 가치는 변하지 않습니다. 오늘은 짧은 안부 문자 한 통을 보내보세요.
```

## 톤 가이드
- 초등형: 매우 쉬운 단어, 1~2문장, 행동 권장 명확  
- 상담형: 공감+설명+권장, 3~4문장  
- 실행형: 위로 짧게 + 구체적 행동 3단계

## 템플릿 매핑(도메인)
- parent: 부모님과의 관계 템플릿  
- spouse: 배우자 템플릿  
- finance: 재정 템플릿  
- career, health, children 등 추후 확장

## 테스트 케이스
- 정통 공망 케이스 4건, 기능적 공백 케이스 4건, 중복 케이스 4건 — 자동 테스트로 검증 예정

## 변경 절차
1. 템플릿 변경 → 2. 샘플 리포트 생성 → 3. 내부 QA(읽기·이해도) → 4. 배포
